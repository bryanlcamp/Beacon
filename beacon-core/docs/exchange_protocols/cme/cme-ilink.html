# CME iLink Protocol Support ## Overview Beacon provides comprehensive support for CME Group's iLink3 execution protocol, enabling high-performance electronic trading on CME exchanges. ## Supported Features ### iLink3 Protocol Version - **Protocol Version**: 8.2 - **Supported Products**: Futures, Options, Spreads - **Session Management**: Binary session protocol - **Order Management**: Full FIFO order book ### Message Types #### Administrative Messages - `Negotiate` - Session establishment - `NegotiateResponse` - Session confirmation - `EstablishmentAck` - Connection acknowledgment - `Sequence` - Message sequencing - `Terminate` - Session termination #### Business Messages - `NewOrderSingle` - Single order entry - `OrderCancelReplaceRequest` - Order modification - `OrderCancelRequest` - Order cancellation - `MassQuote` - Bulk quote submission - `ExecutionReportNew` - Order acknowledgment - `ExecutionReportExecution` - Fill notification ### Performance Characteristics #### Latency Targets - **Order Entry**: < 250μs (gateway to matching engine) - **Fill Reports**: < 100μs (matching engine to client) - **Cancel/Replace**: < 150μs round-trip #### Throughput Limits - **Orders per Second**: 2,000 per session - **Mass Quote Legs**: Up to 20 legs per message - **Session Capacity**: Multiple concurrent sessions ## Configuration ### Session Setup ```json { "execution" : { "protocol" : "CME_ILINK3" , "version" : "8.2" , "host" : "10.1.55.36" , "port" : 9001, "credentials" : { "access_key_id" : "YOUR_ACCESS_KEY_ID" , "session_id" : "SESSION_01" , "firm_id" : "FIRM123" }, "settings" : { "heartbeat_interval" : 30, "test_request_delay" : 60, "session_timeout" : 120 } } } ``` ### Order Management ```cpp // Example order submission CME::iLink3::NewOrderSingle order; order.setClOrdId("ORDER_001"); order.setSide(CME::Side::Buy); order.setOrderQty(100); order.setOrdType(CME::OrdType::Limit); order.setPrice(4150.25); order.setSecurityId(559038); // ES futures session.sendMessage(order); ``` ## Risk Management ### Pre-Trade Controls - **Credit Limits**: Real-time credit checking - **Position Limits**: Maximum position validation - **Rate Limiting**: Order submission throttling - **Product Validation**: Contract specification checks ### Post-Trade Monitoring - **Fill Validation**: Execution price checks - **Position Reconciliation**: Real-time position updates - **P&L Calculation**: Mark-to-market P&L - **Exposure Monitoring**: Portfolio risk assessment ## Market Data Integration ### MDP 3.0 Support - **Incremental Updates**: Real-time order book changes - **Snapshot Recovery**: Full order book reconstruction - **Market Statistics**: Volume, open interest, settlement - **Trade Reports**: Execution price and volume ### Symbol Mapping ```cpp // Map market data to execution symbols struct CMESymbolMap { uint32_t security_id; // MDP security ID std::string symbol; // Human-readable symbol std::string cfi_code; // CFI classification uint32_t market_segment; // Trading venue }; ``` ## Error Handling ### Common Error Codes - `20013` - Insufficient credit - `20025` - Invalid security ID - `20039` - Order quantity too small - `20041` - Price outside valid range - `20043` - Duplicate ClOrdId ### Recovery Procedures 1. **Sequence Gap**: Request retransmission 2. **Session Timeout**: Automatic reconnection 3. **Order Reject**: Parse reject reason and retry 4. **System Maintenance**: Graceful degradation ## Testing & Certification ### CME Test Environment - **Endpoint**: `170.140.30.36:9001` - **Market Data**: `224.0.62.1:14310` - **Test Symbols**: ES, NQ test contracts - **Certification Matrix**: Order types, cancellations, modifications ### Performance Benchmarks ```bash # Run CME-specific performance tests cd tests/protocols ./test_cme_latency --exchange=CME --iterations=100000 # Expected Results: # Order Entry: P50=180μs, P95=250μs, P99=400μs # Fill Report: P50=80μs, P95=120μs, P99=200μs ``` --- **Related**: [iLink Configuration](cme-ilink.md) | [Risk Management](../../reference/risk-management.md) | [Performance Testing](../../reference/performance-testing.md)
