# Beacon Trading Platform Installation Guide ## System Requirements ### Hardware - **CPU**: Intel/AMD x64 with RDTSC support - **Memory**: Minimum 16GB RAM (32GB+ recommended for HFT) - **Network**: Low-latency network interface (10GbE recommended) - **Storage**: SSD for market data and logs ### Software - **Operating System**: Linux (Ubuntu 20.04+ or CentOS 8+) - **Compiler**: GCC 9+ or Clang 10+ with C++17 support - **Build Tools**: CMake 3.20+, Python 3.8+ - **Optional**: Docker for containerized deployment ## Quick Installation ### 1. Clone Repository ```bash git clone https://github.com/bryanlcamp/beacon-core.git cd beacon-core ``` ### 2. Install Dependencies ```bash # Ubuntu/Debian sudo apt update sudo apt install build-essential cmake python3 python3-pip sudo apt install libgtest-dev libbenchmark-dev # CentOS/RHEL sudo yum groupinstall "Development Tools" sudo yum install cmake3 python3 python3-pip ``` ### 3. Build Platform ```bash # Release build (optimized for production) python3 beacon-build.py --release # Debug build (for development) python3 beacon-build.py --debug ``` ### 4. Verify Installation ```bash # Run tests cd build && ctest --verbose # Quick system check python3 beacon-run.py -i config/system/startBeacon.json --duration 10 ``` ## Advanced Configuration ### Performance Optimization ```bash # Enable CPU isolation echo 'isolcpus=2,3' >> /etc/default/grub grub-update && reboot # Set thread affinity in config { "performance": { "thread_affinity_core": 2, "cpu_isolation": true, "memory_locked": true } } ``` ### Network Setup ```bash # Configure multicast for market data ip route add 224.0.0.0/4 dev eth0 echo 'net.core.rmem_max = 134217728' >> /etc/sysctl.conf # Set NIC optimizations ethtool -G eth0 rx 4096 tx 4096 ethtool -K eth0 gro off lro off ``` ## Deployment ### Single-Server Setup ```bash # Production deployment python3 beacon-run.py -i config/system/startBeaconCME.json ``` ### Multi-Server Architecture - **Trading Server**: Client algorithms and execution - **Market Data Server**: Data feed processing - **Matching Engine Server**: Order book management - **Risk Management Server**: Real-time monitoring ## Troubleshooting ### Common Issues **Build Errors** ```bash # Clean rebuild python3 beacon-build.py --clean --release ``` **Permission Errors** ```bash # Set capabilities for network access sudo setcap cap_net_admin+ep ./build/bin/client_algorithm ``` **Latency Issues** - Check CPU affinity settings - Verify network configuration - Monitor system interrupts - Use `perf` for profiling --- **Next Steps**: [System Architecture](system/architecture.md) | [Getting Started](system/getting-started.md)
