Individual Components

Each Beacon module can run independently, allowing you to test specific aspects of your trading infrastructure without setting up a complete pipeline.

  • Configure symbols and parameters

When to Use Stand-Alone Mode

Stand-alone applications are ideal for:

  • Data Preparation: Generate datasets for backtesting or external tools
  • Algorithm Development: Focus on strategy logic without infrastructure
  • Performance Testing: Isolate and optimize individual components
  • Integration Testing: Validate interfaces with existing systems
  • Educational Use: Learn market microstructure concepts

Performance Benefits

Running components independently provides:

  • Resource Isolation: Dedicated CPU and memory per component
  • Simplified Debugging: Focus on single component behavior
  • Flexible Deployment: Mix Beacon components with existing infrastructure
  • Scalable Testing: Run multiple instances simultaneously
