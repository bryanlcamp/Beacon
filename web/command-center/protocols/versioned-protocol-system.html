Versioned Protocol System

Automatic protocol version management and backward compatibility for evolving exchange specifications.

Version Management

Beacon automatically handles protocol versioning to ensure compatibility across exchange updates:

  • Automatic Detection: Protocol version identification from message headers
  • Backward Compatibility: Support for legacy protocol versions
  • Forward Compatibility: Graceful handling of newer protocol features
  • Version Negotiation: Optimal version selection for maximum feature support

Supported Versions

CME Group

  • MDP 3.0: Current production version with full SBE support
  • MDP 2.5: Legacy support for older implementations
  • Future Versions: Ready for upcoming CME protocol updates

Nasdaq

  • ITCH 5.0: Current TotalView protocol version
  • ITCH 4.x: Legacy version compatibility
  • Enhanced Features: Support for new message types as released

NYSE

  • Pillar: Current NYSE trading platform protocol
  • Legacy Systems: Support for transitional periods
  • Market Extensions: Coverage for NYSE American, Arca variants

Implementation Strategy

  • Schema Evolution: Extensible message definitions for new fields
  • Feature Flags: Runtime enablement of version-specific features
  • Testing Matrix: Validation against all supported versions
  • Performance Optimization: Version-specific code paths for maximum speed

Benefits

  • Production Stability: No disruption from exchange protocol updates
  • Development Continuity: Consistent API across protocol versions
  • Migration Support: Smooth transitions during exchange upgrades
  • Future Proofing: Ready for upcoming protocol enhancements

Integration

Version management works transparently with Encoding/Decoding and Protocol Selection systems.
