Choosing a Protocol

Select the optimal protocol for your trading strategy and performance requirements.

Protocol Selection Criteria

Choose the right protocol based on your specific trading needs:

By Asset Class

  • Equity Trading: NASDAQ ITCH for order-by-order data, NYSE Pillar for integrated feeds
  • Futures Trading: CME MDP 3.0 for derivatives and commodities
  • Options Trading: CME or NASDAQ depending on underlying asset
  • Multi-Asset: All protocols supported simultaneously

By Data Requirements

  • Level 1 Data: All protocols provide top-of-book quotes
  • Level 2 Data: Full order book depth available on all exchanges
  • Order-by-Order: NASDAQ ITCH provides granular order tracking
  • Time and Sales: All protocols include trade execution data

By Performance Needs

  • Ultra-Low Latency: CME MDP 3.0 optimized for speed
  • High Throughput: NASDAQ ITCH handles high message volumes
  • Bandwidth Efficiency: All protocols use binary compression
  • Processing Efficiency: Native binary eliminates parsing overhead

Default Recommendations

  • Equity HFT: Start with NASDAQ ITCH for comprehensive order data
  • Futures HFT: Use CME MDP 3.0 for derivatives trading
  • Cross-Asset: Enable multiple protocols simultaneously
  • Backtesting: Match your production environment protocols

Configuration

Protocol selection is configured in the Dataset Generator with automatic optimization for each exchange format.
