System Requirements

Hardware and software specifications for optimal Beacon performance.

Performance Target: 4.5μs Tick-to-Trade

These requirements ensure Beacon delivers production-grade latency with median 4.5μs tick-to-trade and 12μs p99 performance.

Minimum Requirements

Operating System

  • macOS 10.15 (Catalina) or later
  • Ubuntu 20.04 LTS or later
  • CentOS 8 or later
  • Red Hat Enterprise Linux 8+

Hardware

  • 4-core CPU (Intel/AMD x64)
  • 8 GB RAM minimum
  • 10 GB available disk space
  • Network interface (1 Gbps+)

Development Tools

  • C++17 compatible compiler
  • GCC 9.0+ or Clang 10.0+
  • CMake 3.16 or later
  • Git version control

Dependencies

  • Google Test (included)
  • nlohmann JSON (included)
  • Python 3.8+ (optional)
  • Docker (optional)

Recommended for Production

High-Performance Hardware

  • 8+ core CPU with high clock speed
  • 32 GB RAM or more
  • NVMe SSD storage
  • 10 Gbps+ network interface
  • NUMA-aware configuration

Optimization Settings

  • CPU core isolation
  • Real-time kernel (RT_PREEMPT)
  • Disabled power management
  • Optimized network stack
  • Lock-free memory allocation

Platform Support

Current: macOS and Linux with full feature support and optimizations.
Upcoming: Windows implementation in active development.

Performance Considerations

Beacon is designed for maximum performance with minimal system requirements. The platform automatically detects available hardware and optimizes accordingly:

  • → CPU Affinity: Automatic core pinning for critical threads
  • → Memory Management: Lock-free, wait-free data structures
  • → Network Optimization: UDP zero-copy where available
  • → Compiler Optimizations: Profile-guided optimization (PGO)
