Installation Guide

Get Beacon running on your system in minutes with production-grade performance.

Prerequisites

Before installing, ensure your system meets the System Requirements for optimal 4.5μs tick-to-trade performance.

1

Clone the Repository

Download the Beacon source code from the official repository:

git clone https://github.com/bryanlcamp/beacon.git cd beacon

This will create a local copy of the Beacon platform with all source code, documentation, and build scripts.

2

Install Dependencies

Install build tools using Homebrew:

# Install Homebrew if not already installed /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Install required dependencies brew install cmake gcc git python3

Install build tools using apt:

# Update package manager sudo apt update # Install required dependencies sudo apt install -y build-essential cmake git python3 python3-pip

Install build tools using yum/dnf:

# For CentOS 8+ / RHEL 8+ sudo dnf groupinstall -y "Development Tools" sudo dnf install -y cmake git python3 # For CentOS 7 / RHEL 7 sudo yum groupinstall -y "Development Tools" sudo yum install -y cmake3 git python3
3

Build Beacon

Compile Beacon with optimizations for maximum performance:

# Create build directory mkdir build && cd build # Configure with release optimizations cmake .. -DCMAKE_BUILD_TYPE=Release -DENABLE_OPTIMIZATIONS=ON # Build (use all available cores) make -j$(nproc)

Performance Tip

The Release build includes profile-guided optimization (PGO) and CPU-specific optimizations for maximum performance.

4

Verify Installation

Run the test suite to ensure everything is working correctly:

# Run unit tests make test # Run performance benchmarks ./bin/beacon_benchmark # Start the web interface ./bin/beacon_server --port 8080

If all tests pass, navigate to http://localhost:8080 to access the Beacon web interface.

5

Configure for Production

For production deployments, optimize system settings:

# Set CPU governor to performance echo performance | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor # Disable CPU power management sudo systemctl mask sleep.target suspend.target # Configure network optimizations sudo sysctl -w net.core.rmem_max=134217728 sudo sysctl -w net.core.wmem_max=134217728

Production Note

These optimizations require root access and will affect system-wide performance. Review your organization's policies before applying.

Quick Start

Once installed, you can immediately start testing algorithms:

# Generate sample market data ./bin/dataset_generator --symbol AAPL --duration 60m --output sample_data.bin # Start market data playback ./bin/market_data_playback --input sample_data.bin --protocol nasdaq # Connect your algorithm (in another terminal) ./bin/your_algorithm --connect localhost:5555

Troubleshooting

  • → Build Errors: Ensure GCC 9.0+ or Clang 10.0+ is installed
  • → Permission Issues: Run with appropriate sudo privileges for system optimizations
  • → Performance Issues: Verify system meets minimum requirements
  • → Network Issues: Check firewall settings for UDP/TCP ports
