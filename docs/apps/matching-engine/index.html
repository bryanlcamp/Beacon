Matching Engine

High-performance order matching and execution simulation

Overview

The Matching Engine simulates realistic exchange order matching behavior, providing a complete order book implementation with price-time priority matching. It processes incoming orders and generates execution reports using native exchange protocols.

Key Features

  • Price-time priority order matching algorithm
  • Multi-protocol support (NASDAQ OUCH, NYSE Pillar, CME iLink)
  • Realistic partial fill simulation
  • Order book depth tracking and visualization
  • Configurable latency and rejection simulation

Usage

The matching engine runs as a TCP server, accepting order entry connections from trading algorithms. It processes orders in real-time and sends back execution reports using protocol-specific binary messaging.

Configuration

Engine behavior is configured through JSON files that specify protocol settings, port bindings, market hours, and matching parameters. Different configurations are available for each supported exchange protocol.
