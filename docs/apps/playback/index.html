Playback Engine

Historical market data replay for algorithm backtesting

Overview

The Playback Engine replays historical market data files to test trading algorithms against real market conditions. It supports multiple data formats and can replay data at various speeds, allowing for comprehensive backtesting and algorithm validation.

Key Features

  • Historical data file replay (PCAP, binary feeds)
  • Multiple exchange data format support
  • Configurable playback speed (1x to 1000x real-time)
  • Date range and symbol filtering
  • Integration with matching engine for realistic execution

Usage

The playback engine loads historical data files and replays them over UDP, mimicking real-time market data feeds. Algorithms can connect and trade against this replayed data to validate performance and behavior.

Configuration

Playback sessions are configured through JSON files that specify data file locations, playback parameters, symbol filters, and output settings. The engine supports both full-day replays and specific time window testing.
