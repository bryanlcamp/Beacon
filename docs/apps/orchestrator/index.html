Orchestrator

System coordination and component lifecycle management

Overview

The Orchestrator manages the lifecycle and coordination of all Beacon components. It handles component startup sequencing, health monitoring, graceful shutdown, and inter-component communication coordination to ensure the entire trading system operates as a cohesive unit.

Key Features

  • Component lifecycle management (start, monitor, stop)
  • Dependency-aware startup sequencing
  • Health monitoring and restart capabilities
  • Configuration distribution and validation
  • Centralized logging and error reporting

Usage

The orchestrator is typically launched by beacon-run.py and operates behind the scenes to coordinate all system components. It reads system configuration files and manages the complete trading session lifecycle.

Configuration

Orchestrator behavior is controlled through system-level JSON configuration files that define component dependencies, startup sequences, monitoring parameters, and error handling policies.
