Prerequisites

Before installing and running Beacon, ensure your system meets the following requirements and has the necessary dependencies installed.

System Requirements

Operating System
Linux (Ubuntu 20.04+) or macOS (10.15+)

Beacon is optimized for Unix-like systems with low-latency networking capabilities.

uname -a
CPU Architecture
x86_64 (Intel/AMD) or ARM64 (Apple Silicon)

Multi-core processor recommended for optimal performance with parallel processing.

lscpu | grep Architecture
Memory
Minimum: 8 GB RAM, Recommended: 16+ GB

Higher memory allows for larger market data datasets and multiple concurrent algorithms.

free -h
Storage
Minimum: 10 GB free space, Recommended: SSD

SSD storage significantly improves build times and data loading performance.

df -h

Platform-Specific Installation

Ubuntu/Debian

# Update package manager
sudo apt update

# Install build tools
sudo apt install -y build-essential cmake git

# Install additional tools
sudo apt install -y ninja-build pkg-config

# Verify installation
cmake --version && gcc --version && git --version

macOS

# Install Xcode Command Line Tools
xcode-select --install

# Install Homebrew (if not already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install dependencies
brew install cmake git ninja

# Verify installation
cmake --version && clang --version && git --version

CentOS/RHEL/Fedora

# CentOS/RHEL
sudo yum groupinstall "Development Tools"
sudo yum install cmake git

# Fedora
sudo dnf groupinstall "Development Tools"
sudo dnf install cmake git ninja-build

# Verify installation
cmake --version && gcc --version && git --version

Network Configuration

⚠️ Network Requirements:
Beacon uses UDP multicast and TCP connections for market data and order flow. Ensure your firewall and network configuration allow these protocols on the configured ports (default: 8000-9000 range).

Firewall Configuration

# Ubuntu/Debian - Allow Beacon ports
sudo ufw allow 8000:9000/tcp
sudo ufw allow 8000:9000/udp

# CentOS/RHEL/Fedora - Allow Beacon ports
sudo firewall-cmd --permanent --add-port=8000-9000/tcp
sudo firewall-cmd --permanent --add-port=8000-9000/udp
sudo firewall-cmd --reload

Loopback Interface

Beacon can run entirely on localhost for development and testing:

# Test loopback networking
ping 127.0.0.1

# Verify multicast support
ip maddr show

Performance Optimization (Optional)

For Production Deployments:
These optimizations are recommended for low-latency trading environments but not required for development or testing.

CPU Isolation

# Reserve CPU cores for Beacon (add to /etc/default/grub)
GRUB_CMDLINE_LINUX="isolcpus=2,3,4,5"

# Update GRUB and reboot
sudo update-grub && sudo reboot

Memory Settings

# Increase shared memory limits (add to /etc/sysctl.conf)
kernel.shmmax = 17179869184
kernel.shmall = 4194304

# Apply settings
sudo sysctl -p

Network Tuning

# Increase network buffer sizes (add to /etc/sysctl.conf)
net.core.rmem_max = 134217728
net.core.wmem_max = 134217728
net.ipv4.udp_mem = 102400 873800 16777216

# Apply settings
sudo sysctl -p

Verification Script

Run this script to verify your system meets all prerequisites:

#!/bin/bash
# Beacon Prerequisites Check

echo "=== Beacon Prerequisites Check ==="
echo

# Check OS
echo "Operating System:"
uname -a
echo

# Check CPU
echo "CPU Information:"
lscpu | grep -E "(Architecture|CPU|Model)"
echo

# Check Memory
echo "Memory:"
free -h
echo

# Check Storage
echo "Storage:"
df -h /
echo

# Check Build Tools
echo "Build Tools:"
cmake --version 2>/dev/null || echo "❌ CMake not found"
gcc --version 2>/dev/null | head -1 || echo "❌ GCC not found"
make --version 2>/dev/null | head -1 || echo "❌ Make not found"
git --version 2>/dev/null || echo "❌ Git not found"
echo

echo "✅ Prerequisites check complete!"
echo "If any tools are missing, install them using the platform-specific instructions above."
✅ Ready to Proceed:
Once all prerequisites are met, you can proceed to the Getting Started guide for installation and setup instructions.

Troubleshooting

Common Issues

CMake Version Too Old

Error: CMake 3.16 or higher is required

Solution: Install a newer version from the official CMake website or use a newer Linux distribution.

Compiler Not Found

Error: No CMAKE_CXX_COMPILER could be found

Solution: Install build-essential (Ubuntu) or Development Tools (CentOS/Fedora).

Permission Denied (Network Ports)

Error: bind: Permission denied

Solution: Check firewall settings or use ports > 1024 for non-root execution.

Multicast Not Working

Error: Market data not received

Solution: Verify multicast routing: ip route show and ensure loopback interface supports multicast.
