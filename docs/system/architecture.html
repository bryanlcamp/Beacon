Overview

The Beacon Trading System is a complete, multi-protocol electronic trading ecosystem designed for ultra-low latency algorithmic trading. The system supports multiple exchange protocols (NASDAQ OUCH, NYSE Pillar, CME iLink 3) and provides end-to-end infrastructure for market data generation, distribution, order execution, and algorithmic trading.

System Architecture Flow

Market Data Generator → UDP Multicast → Algorithm
                                                        ↓
Algorithm ← TCP Orders ← Matching Engine ← TCP Orders

System Components

1. Market Data Generator

Purpose: Generates synthetic market data in exchange-native formats

Key Features:

  • Multi-protocol support: NASDAQ ITCH, NYSE Pillar, CME MDP
  • Configurable symbol behavior and price ranges
  • Realistic market microstructure simulation
  • High-frequency message generation (100K+ msgs/sec)

Communication:

  • Output: UDP multicast (239.255.0.1:12345)
  • Protocols: ITCH 5.0, Pillar, MDP 3.0

View detailed documentation →

2. Market Data Playback

Purpose: Replays pre-recorded or synthetic market data for testing and backtesting

Key Features:

  • File-based playback with speed control
  • UDP multicast streaming
  • Timestamped message replay
  • Loop/repeat functionality

View detailed documentation →

3. Exchange Matching Engine

Purpose: Simulates exchange order matching and execution

Key Features:

  • Multi-protocol support (OUCH, Pillar, iLink 3)
  • Price-time priority matching algorithm
  • Real-time order book maintenance
  • Fill generation and reporting

Performance:

  • Order processing latency: < 5 μs
  • Throughput: 100,000+ orders/sec
  • Lock-free order processing

View detailed documentation →

4. Client Algorithm

Purpose: Algorithmic trading client with comprehensive risk management

Key Features:

  • Real-time market data processing
  • Multi-protocol order generation
  • Advanced risk management layers
  • Position and PnL tracking

Risk Management:

  • PnL-based limits (Warning/Alert/Hard Stop)
  • Order rejection monitoring
  • Burst rate control
  • Position limits (per-product and firm-wide)

5. Orchestrator

Purpose: System coordination and component lifecycle management

Key Features:

  • Component lifecycle management

Communication Flow

Complete Trading Loop

  1. Market Data Generator → UDP Multicast
  2. Algorithm receives market data (< 1 μs)
  3. Algorithm makes trading decision (< 5 μs)
  4. Algorithm sends order via TCP (< 10 μs)
  5. Matching Engine receives and matches order (< 5 μs)
  6. Matching Engine sends fill report (< 5 μs)
  7. Algorithm receives fill, updates position/PnL (< 2 μs)

Total Round-Trip Latency Target: < 30 μs

Building & Running

Quick Build

python3 beacon-build.py --release

Running Complete System

# Start all components
python3 beacon-run.py -i config/system/startBeacon.json

# The system will coordinate:
# 1. Matching engine startup
# 2. Market data generation 
# 3. Algorithm connection
# 4. Trading session monitoring

Manual Component Startup

# Terminal 1: Start Matching Engine
cd build/src/apps/matching_engine
./matching_engine

# Terminal 2: Start Market Data Generator  
cd build/src/apps/generator
./generator --config config.json

# Terminal 3: Start Algorithm
cd build/src/apps/client_algorithm
./client_algo --config sample_config.json

Architecture Highlights

Design Principles

  • Lock-Free Design: SPSC queues, ring buffers, zero-copy techniques
  • Memory Efficiency: Pre-allocated buffers, object pooling, cache-aligned structures
  • Network Optimization: UDP multicast, non-blocking I/O, kernel bypass options
  • Observability: Built-in latency instrumentation, configurable logging, performance counters
