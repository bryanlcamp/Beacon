Quick Reference

Essential commands and configuration guide

Build Commands

Build Everything

python3 beacon-build.py

CMake Build (Manual)

cmake -B build -S .
cmake --build build

Run Commands

Full System (Automated)

# Complete system
python3 beacon.py

# Or use run script
python3 beacon-run.py

Manual Launch (3 terminals)

Terminal 1: Matching Engine

./build/src/apps/matching_engine/matching_engine

Terminal 2: Market Data

# Generator
./build/src/apps/generator/generator

# OR Playback
./build/src/apps/playback/playback

Terminal 3: Trading Algorithm

./build/src/apps/client_algorithm/AlgoTwapProtocol

Performance Monitoring

Check Latency

# Add to any component
auto start = std::chrono::high_resolution_clock::now();
// ... operation ...
auto end = std::chrono::high_resolution_clock::now();
auto latency_us = std::chrono::duration_cast<std::chrono::microseconds>(end - start).count();

Monitor Network Traffic

# UDP multicast (market data)
sudo tcpdump -i any udp port 8002 -v

# TCP orders
sudo tcpdump -i any tcp port 9002 -v

# Packet rates
iftop -i lo -f "port 8002 or port 9002"

System Performance

# CPU usage
top -p $(pgrep -d',' matching_engine)

# Profile with perf (Linux)
perf record -g ./build/src/apps/matching_engine/matching_engine
perf report

Protocol Selection

NASDAQ OUCH

  • Market data: NASDAQ ITCH 5.0
  • Orders: OUCH 5.0
  • Port: 9002

NYSE Pillar

  • Market data: NYSE Pillar
  • Orders: Pillar Gateway 3.2
  • Port: 9002

CME iLink 3

  • Market data: CME MDP 3.0
  • Orders: iLink 3
  • Port: 9002

Note: Matching engine auto-detects protocol based on incoming message format.

Troubleshooting

No Market Data

# Check if generator is running
ps aux | grep generator

# Test UDP reception
nc -u -l 8002

Orders Not Executing

# Check if matching engine is running
ps aux | grep matching_engine

# Test TCP connection
telnet 127.0.0.1 9002

# Check port binding
netstat -an | grep 9002

Algorithm Not Trading

# Check configuration
cat beacon-config.json

# Verify settings are valid
python3 beacon-config.py --validate

Testing Checklist

  • Build all applications successfully
  • Launch matching engine (port 9002 listening)
  • Start market data source (UDP)
  • Start algorithm (connects to both)
  • Verify market data reception
  • Verify order submission
  • Verify fill reception
  • Check latency metrics
  • Test risk limit triggers
  • Test position limit enforcement
  • Test protocol switching (OUCH, Pillar, iLink 3)

Performance Targets

Metric Target
Market data processing < 1 μs
Trading decision < 5 μs
Order submission < 10 μs
Order matching < 5 μs
Fill processing < 2 μs
Total round-trip < 30 μs
Throughput > 100,000 orders/sec

Log Files

Logs are written to the console by default. To redirect:

python3 beacon.py > beacon.log 2>&1

To monitor in real-time:

tail -f beacon.log

Emergency Shutdown

Graceful

# If using beacon.py
Ctrl+C (in beacon.py terminal)

# Manual
pkill -SIGTERM matching_engine
pkill -SIGTERM generator
pkill -SIGTERM AlgoTwapProtocol

Force

pkill -9 matching_engine
pkill -9 generator
pkill -9 AlgoTwapProtocol
