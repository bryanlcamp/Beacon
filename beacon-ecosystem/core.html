Overview

Core is Beacon's ultra-low latency trading engine designed to execute strategies with Wall Street-grade performance without requiring co-location. It combines zero-copy messaging, NUMA-aware memory allocation, and hardware-optimized networking to achieve sub-microsecond execution latencies on standard server hardware.

Core Capabilities

Sub-Microsecond Execution

Strategy execution with 3.1μs P99 latency from market data receipt to order transmission.

NUMA-Aware Architecture

Intelligent core pinning and memory allocation that maximizes cache efficiency and minimizes latency variance.

Zero-Copy Messaging

Direct memory access patterns that eliminate unnecessary data copying and reduce memory pressure.

Protocol Optimization

Native support for ITCH/OUCH protocols with hardware-accelerated message parsing.

Technical Architecture

Hardware Independence

Unlike traditional HFT platforms that require specialized hardware or co-location, Core achieves exceptional performance on standard server hardware. Deploy anywhere with confidence that latency characteristics remain consistent.

Memory Management

Core employs sophisticated memory management techniques:

  • Pre-allocated object pools to eliminate garbage collection pauses
  • Lock-free data structures for concurrent strategy execution
  • CPU cache-aware data layout optimization
  • Huge page support for reduced TLB pressure

Network Stack

The networking subsystem bypasses kernel overhead through:

  • User-space TCP/UDP implementation
  • Direct NIC access via DPDK integration
  • Batched message processing with flow control
  • Hardware timestamping for precise latency measurement

Strategy Development API

Core provides a clean, high-performance API that abstracts low-level optimizations while exposing the necessary controls for sophisticated strategies:

Market Data Interface

  • Real-time order book reconstruction with depth visibility
  • Trade and quote stream processing with microsecond timestamps
  • Custom aggregation and windowing functions

Order Management

  • Direct market access with venue-specific order types
  • Portfolio-level risk controls and position tracking
  • Real-time P&L calculation and exposure monitoring

Execution Controls

  • Configurable execution policies and routing logic
  • Dynamic venue selection based on liquidity and latency
  • Intelligent order slicing and iceberg execution

Performance Characteristics

Core's performance profile enables strategies that were previously only viable with co-located infrastructure:

  • Market Data Processing: Sub-100ns message parsing
  • Strategy Computation: ~500ns for typical alpha generation
  • Order Generation: ~200ns order creation and validation
  • Network Transmission: ~2μs to wire with hardware timestamping

Integration with Beacon Ecosystem

Core seamlessly integrates with other Beacon components:

  • Pulse delivers market data with production-accurate timing characteristics
  • Match provides realistic execution simulation including venue-specific latency modeling
  • Author generates the synthetic datasets that test strategy robustness

Deployment Options

Cloud Deployment

Core runs efficiently on cloud platforms including AWS, Azure, and GCP. Specific instance types are optimized for different strategy profiles and latency requirements.

On-Premises

For maximum control and minimal latency variance, Core deploys on dedicated hardware with optimized kernel configurations and network tuning.

Hybrid Configurations

Development and testing in cloud environments with production deployment on optimized hardware, maintaining identical API interfaces across environments.

Core represents a fundamental shift in trading platform architecture: achieving co-location-grade performance without the geographic and infrastructure constraints. This democratizes access to ultra-low latency trading while maintaining the precision required for institutional-grade strategies.
